# 为什么你的 Agent 总是"失忆"？这篇把记忆系统讲透了

---

想象一下这个场景。

你招了一个超级聪明的员工。第一天上班，她表现炸裂——抓 bug、写文档、提建议，样样精通。你心想：这波赚到了。

第二天，你走进办公室："嘿，昨天那个问题我们怎么说的来着？"

她停顿了一下。看着你。微微一笑。

"抱歉……什么问题？"

**没有记忆。没有上下文。完全消失。**

你会怎么想？疯了，对吧？

但这正是大多数 LLM 的真实状态。每一次新对话，都是全新开始。它不知道你是谁、你们一起做过什么、上一秒在另一个窗口聊了什么。

**对于聊天机器人，这没问题。但对于 Agent——一个要执行任务、做决策、持续改进的系统——这种"失忆症"是致命的。**

因为真正的智能不只是"回答得好"。它是**记住、学习、在前人基础上构建**。

**记忆，是把无状态系统变成能真正进化的东西的关键。**

---

## 一、Agent 记忆到底在干什么？

很多人以为"给 Agent 加个数据库"就是记忆系统了。

**远远不够。**

Agent 记忆不是做一件事，而是**同时做三件截然不同的事**：

| 功能 | 解决什么问题 | 没有它会怎样 |
|-----|-------------|-------------|
| **连续性** | Agent 知道你是谁、你偏好什么 | 每次对话都像第一次见面 |
| **上下文** | 当前任务状态、工具调用历史 | 多步骤工作流直接崩溃 |
| **学习** | 从成功/失败中改进决策 | 同样的错误反复犯 |

**把这三件事都做好，Agent 才会让人觉得"靠谱"、"聪明"、"越用越顺手"。**

---

## 二、四种记忆类型：每种都有特定用途

研究界已经达成共识：Agent 记忆分四种类型。可以把它们想象成大脑的四个"分区"，每个分区进化出来就是为了干特定的活。

### 2.1 In-Context Memory：工作台记忆

**上下文窗口就是 Agent 的办公桌**。桌上的一切都可以立即访问，模型可以在一次前向传递中对它们进行推理。不需要检索步骤。

**但办公桌有大小限制**。每个 token 都要花钱和时间。会话结束，桌子就被清空。

**桌上放什么？**

| 内容 | 作用 |
|-----|------|
| System Prompt | Agent 人格、规则、能力 |
| 对话历史 | 本次会话的来来回回 |
| 工具调用结果 | 刚刚调用的工具返回了什么 |
| 检索到的记忆 | 从外部存储拉进来的片段 |
| Scratchpad | 中间推理（"让我们一步步思考"） |

**核心问题：滑动窗口**

长对话中，历史会累积并最终溢出上下文限制。简单的"截断最旧消息"方案会丢失重要信息。更好的策略：

- **摘要**：定期把旧对话压缩成简短摘要
- **选择性保留**：保留关键事实和决策，丢弃闲聊
- **卸载到外部记忆**：提取重要事实到向量存储，按需检索

---

### 2.2 External Memory：外部持久化记忆

外部记忆是在模型之外持久化的任何东西——数据库、向量存储、键值存储、文件。它会**跨会话存活**。存得好，你的 Agent 能记住六个月前的事。

**两种外部存储方式：**

| 类型 | 工具 | 查询方式 |
|-----|------|---------|
| **结构化存储** | PostgreSQL, Redis, SQLite | 按 key、ID 或 SQL 查询。快、可预测 |
| **向量存储** | Pinecone, Chroma, pgvector | 按意义查询，"找相似概念的记忆" |

**关键洞察：检索是瓶颈**

如果你没检索到正确的记忆，Agent 就会表现得好像它们不存在。

**好的记忆架构 = 20% 存储 + 80% 检索设计。**

---

### 2.3 Episodic Memory：情景记忆

**这是最被低估的类型。**

外部记忆存储"事实"，情景记忆存储"事件"——具体来说，是过去行动的**结果**。

最简单的形式是一个结构化日志：每次 Agent 完成任务，它记录发生了什么。久而久之，这个日志成为 Agent 在做决策前可以查阅的"自我知识库"。

**一个 Episode 长这样：**

```json
{
  "task": "把 50 页 PDF 总结成 3 个要点",
  "approach": "顺序分块，每块 2000 tokens",
  "outcome": "success",
  "duration_ms": 4820,
  "token_cost": 12400,
  "quality_score": 0.91,
  "notes": "效果好，分层分块会更快"
}
```

**新任务进来时，Agent 检索语义上最相似的过去 episodes，用它们来选择策略。**

这本质上是从个人历史中做 few-shot 学习——不是从手工制作的数据集，而是从自己的经验中学习。

---

### 2.4 Parametric Memory：参数记忆

**这是模型"与生俱来"的记忆。** 训练时编码到权重里的一切——世界知识、语言模式、推理策略、编程规范、文化常识。

它永远存在，Agent 不需要检索它。但有硬性限制：

| 限制 | 说明 |
|-----|------|
| 训练时冻结 | 模型不知道截止日期后发生的事 |
| 运行时无法更新 | 不重新训练就不能注入新事实 |
| 不透明 | 你无法检查模型"知道"什么 |
| 容易产生幻觉 | 模型用看似合理的错误填补空白 |

**正确的心智模型**：

> 参数记忆是 Agent 的"通识教育"。外部记忆、情景记忆、上下文记忆是 Agent 的"在职经验"。最好的 Agent 组合使用两者。

## 三、记忆如何在 Agent 循环中流动？

把所有东西放在一起。每次 Agent 处理请求，都会发生这样的流程：

```
用户请求
    │
    ▼
┌─────────────────────────────┐
│  Step 1: 检索记忆            │
│  • 查询向量存储获取相关记忆   │
│  • 查询情景记忆获取相似任务   │
│  • 构建注入上下文            │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│  Step 2: 构建输入            │
│  • System Prompt + 人格     │
│  • 注入的记忆上下文          │
│  • 当前用户消息              │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│  Step 3: LLM 调用（无状态）  │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│  Step 4: 写入记忆            │
│  • 提取事实 → 外部记忆       │
│  • 记录 episode → 情景记忆   │
│  • 更新历史 → 上下文记忆     │
└─────────────────────────────┘
    │
    ▼
返回响应
```

**关键洞察**：记忆操作包围 LLM 调用——之前检索，之后写入。模型本身是无状态的；记忆系统是赋予"有状态、有意识 Agent"幻觉的东西。

## 四、记忆管理：不只是存，还得忘

真正的记忆系统不只是累积。它还会**策展**。

一个不断增长、没有焦点的存储会随时间退化——检索变嘈杂、延迟增加、矛盾的记忆让 Agent 困惑。

**你需要一个遗忘策略。**

### 4.1 时间衰减

旧记忆不太相关。通过相关性和时间戳的组合评分：

```python
def memory_score(relevance, importance, created_at):
    hours_old = (now - created_at).hours
    recency = 0.995 ** hours_old  # 衰减因子
    
    return relevance * 0.4 + importance * 0.3 + recency * 0.3
```

灵感来自 Generative Agents 论文（Park et al., 2023）。

### 4.2 写入时重要性评分

存储记忆前，让 LLM 为信息评分重要性。只存高分项目。从源头过滤噪音。

```python
async def score_importance(content):
    prompt = f"""评估保存此信息的重要性（0.0-1.0）
    0.0 = 琐碎（问候）
    0.5 = 中等有用
    1.0 = 关键（偏好、错误、决策）
    
    信息：{content}
    只回复数字。"""
    
    return await llm.ask(prompt)
```

### 4.3 定期合并

运行夜间作业，把重复或高度相似的记忆合并为单个摘要。这类似于人类睡眠如何巩固记忆。

## 五、笔者的判断：记忆层是 Agent 的"第二大脑"

读完这篇，我最大的感受是：**记忆不是锦上添花，是 Agent 的基础设施**。

没有记忆的 Agent，就像一个每天醒来都"重启"的员工——再聪明也没法积累。

**四种记忆类型各有定位**：
- In-Context：工作台，即时可用，会话清空
- External：持久化，跨会话存活，需要检索
- Episodic：经验库，从自己的历史中学习
- Parametric：通识教育，训练时冻结

**真正难的不是存，是检索和管理**：
- 检索是瓶颈：80% 的设计精力应该在这里
- 遗忘很重要：不管理的记忆系统会退化
- 学习是目标：记忆最终是为了让 Agent 变强

**不得不感叹一句：把记忆层设计好，Agent 的其他一切都会变得更聪明。**
