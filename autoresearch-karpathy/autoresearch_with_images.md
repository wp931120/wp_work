## Hook：你有没有想过，AI 也能自己搞研究了？

想象一下这个场景：你晚上睡觉前启动一个程序，AI 智能体开始自主设计实验、修改代码、训练模型、验证结果。它保留有效的改进，丢弃无效的尝试，然后继续下一轮。第二天早上醒来，你看到它已经跑了 100 个实验，模型效果也提升了。

这不是科幻，这是 Karpathy 这周刚开源的 **AutoResearch**。

更狠的是，整个系统只有**3 个核心文件**，AI 只能修改其中 1 个。

---

## 一、这是个啥？为什么现有方案不够好？

所谓 AutoResearch，其实就像一个**不知疲倦的研究生**——你给它一个研究方向，它能自己设计实验、跑代码、看结果、调整方案，然后无限循环。

**它解决了什么问题？**

LLM 训练调参是个苦差事。模型结构、学习率、优化器、数据配比……每个维度都有无数种组合。传统做法要么靠经验猜，要么暴力网格搜索，要么干脆不管。

**为什么现有方案不够好？**

- 网格搜索太费钱，几百个实验跑不起
- 随机搜索效率低，大量计算浪费在明显不好的组合上
- 人工调参上限低，还容易陷入局部最优

AutoResearch 的思路是：**让 AI 来帮你试**。但它不是漫无目的地试，而是在一个精心设计的"沙盒"里，用固定时间预算（5 分钟）快速验证每个想法。

## 二、核心架构：3 个文件，1 个约束

这个项目的设计哲学让笔者印象深刻：**极简到几乎苛刻**。

整个系统只有 3 个核心文件：

| 文件 | 用途 | 能否修改 |
|------|------|----------|
| `prepare.py` | 数据准备、分词器、评估函数 | ❌ 固定 |
| `train.py` | 模型定义、优化器、训练循环 | ✅ AI 可改 |
| `program.md` | AI 智能体的操作指令 | ✅ 人类可改 |

**核心约束**：AI 只能修改 `train.py`，不能动数据准备和评估逻辑。

这个设计非常聪明——它保证了评估的公平性，同时给 AI 留出了足够的实验空间（模型结构、优化器、超参数等）。

### 2.1 训练系统架构（`train.py`）

模型是基于 GPT 的 Decoder-only 架构，但加入了一些现代技术：

- **滑动窗口注意力**（SSSL 模式）：减少计算量
- **Value Embedding**（ResFormer）：提升长序列建模能力
- **旋转位置编码**（RoPE）：标准配置
- **RMSNorm 归一化**：训练更稳定

优化器是自定义的 **MuonAdamW**，混合了两种优化策略：
- Muon 优化器用于 2D 矩阵参数（带正交化）
- AdamW 用于嵌入层和标量参数

### 2.2 数据准备（`prepare.py`）

这部分是固定的，AI 不能改：
- 从 HuggingFace 下载 ClimbMix-400B 数据集
- 使用 rustbpe 训练 BPE 分词器（词表大小 8192）
- BOS-aligned 数据打包，100% 利用率（无 padding）

### 2.3 AI 智能体指令（`program.md`）

这是项目的灵魂所在。`program.md` 定义了 AI 的实验循环：

```
1. 查看当前 git 状态
2. 修改 train.py 进行实验
3. git commit 保存改动
4. 运行训练（5 分钟）
5. 读取验证结果（val_bpb，越低越好）
6. 如果改进了就保留，否则回退
7. 记录到 results.tsv
8. 回到第 1 步，无限循环...
```

---

## 三、5 分钟实验循环：这才是关键创新

笔者觉得这个项目最厉害的设计是**固定时间预算**。

所有实验都只跑 5 分钟（墙钟时间），然后看验证集效果。改进了就保留，没改进就回退。这样设计有几个好处：

1. **结果可直接比较**：都是 5 分钟，公平对比
2. **自动适应硬件**：不同 GPU 都能跑，只是步数不同
3. **快速迭代**：一晚上能跑约 100 个实验
4. **成本可控**：不会无限烧钱

评估指标是 **val_bpb**（validation bits per byte），越低越好。这个指标比 perplexity 更直观——它表示模型压缩数据的能力。

## 四、超参数空间：AI 能改什么？

AI 可以在 `train.py` 里修改这些超参数：

```python
# 模型架构
ASPECT_RATIO = 64           # model_dim = depth * ASPECT_RATIO
HEAD_DIM = 128              # 注意力头维度
WINDOW_PATTERN = "SSSL"     # 滑动窗口模式

# 优化器配置
TOTAL_BATCH_SIZE = 2**19    # ~524K tokens/step
EMBEDDING_LR = 0.6
UNEMBEDDING_LR = 0.004
MATRIX_LR = 0.04            # Muon 学习率
WEIGHT_DECAY = 0.2
WARMUP_RATIO = 0.0
WARMDOWN_RATIO = 0.5

# 模型大小
DEPTH = 8                   # Transformer 层数
```

AI 可以尝试修改这些值，看看哪个组合能让 val_bpb 更低。理论上，它可能发现人类没想到的配置。

## 五、使用方式：10 分钟上手

```bash
# 1. 安装 uv（Python 包管理器）
curl -LsSf https://astral.sh/uv/install.sh | sh

# 2. 克隆项目并安装依赖
git clone https://github.com/karpathy/autoresearch
cd autoresearch
uv sync

# 3. 下载数据和训练分词器
uv run prepare.py

# 4. 手动跑一次训练验证
uv run train.py

# 5. 启动 AI 智能体
# 在 Claude/Codex 中打开 program.md，让它开始自主实验
```

## 六、笔者的判断：看好，但有前提

这个项目笔者仔细扒了源码，说说笔者的看法。

**看好的理由：**

1. **设计哲学对路**：用约束换自由，用固定时间换公平对比
2. **可扩展性强**：今天是 3 个文件，明天可以扩展到 10 个
3. **社区价值大**：开源后大家都能贡献实验结果，加速研究
4. **方向正确**：自主 AI 研究是未来，这个是很好的原型

**需要警惕的点：**

1. **5 分钟可能不够**：有些改进需要更长时间训练才能显现
2. **单指标局限**：val_bpb 不是唯一指标，可能忽略其他重要维度
3. **计算成本**：虽然可控，但 100 个实验也不是小数目
4. **泛化能力**：在 ClimbMix-400B 上有效，不代表在其他数据集上也有效

**笔者的建议：**

如果你在做 LLM 预训练相关研究，这个项目值得 fork 下来试试。即使不用它做实验，它的设计思路（固定时间预算、透明追溯、极简约束）也值得借鉴。

## 七、延伸思考：自主 AI 研究的未来

Karpathy 这个项目让笔者想到一个更大的问题：**AI 自主研究的边界在哪里？**

现在的 AutoResearch 还很简单：
- 只能改一个文件
- 只有一个评估指标
- 实验空间相对有限

但想象一下，如果把这个思路扩展：
- AI 可以修改数据预处理逻辑
- AI 可以设计新的模型架构
- AI 可以同时优化多个目标
- AI 可以阅读论文并提出新假设

那时候，AI 就不再是"工具"，而是真正的"研究合作者"。

当然，这也会带来新的问题：如何评估 AI 提出的想法？如何避免 AI 陷入局部最优？人类研究者应该扮演什么角色？

## 结语

AutoResearch 是一个精彩的实践，它展示了如何用极简的设计实现自主 AI 研究系统。

**核心 takeaway**：
- 固定时间预算是关键创新
- 约束不是限制，而是为了让实验更公平
- 开源设计让社区能共同加速研究

最后，送上一句 Karpathy 在 README 里的话：

> "用户晚上睡觉时，AI 可以自主运行约 100 个实验。早上醒来，你就能看到实验日志和（希望）更好的模型。"

这大概就是 AI 时代的研究者该有的生活方式吧。
