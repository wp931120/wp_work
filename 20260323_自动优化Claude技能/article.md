# 你的 Claude Skills 可能正在悄悄失败 30%

> 如何用 Karpathy 的 autoresearch 方法，让你的 AI 技能自动进化

---

你有没有发现，同一个 Claude Skill，有时候输出惊艳，有时候却一塌糊涂？

问题不在模型，而在你的提示词——它可能只在你"运气好"的时候才管用。

今天分享一个来自 Andrej Karpathy（OpenAI 联合创始人、特斯拉前 AI 负责人）的方法，可以让你的任何 Skill 自动迭代优化，完全不用你动手调参。

---

## 这个方法从哪来？

Karpathy 最近发布了一个叫 **autoresearch** 的方法。

核心思想很简单：

> 不用手动改进，让 AI Agent 在循环中自己试错、自己优化。

它的工作原理是：尝试一个小改动 → 检查结果是否变好 → 变好了就保留，变差了就撤销 → 然后再试下一个改动。一遍又一遍。

Karpathy 最初用它优化机器学习代码。但这个方法的本质是：**任何可以被量化、被改进的东西，都可以用 autoresearch。**

包括你写的 Claude Skills。

---

## 一个类比：你有一个"不太稳定"的菜谱

想象你有一个菜谱，10 次里有 7 次做出来很棒，但另外 3 次总是差点意思——可能酱汁太淡，可能调味不对。

与其从头重写整个菜谱，你选择：

1. 改变一个配料
2. 做 10 次试试
3. 结果变好了？保留这个改动
4. 结果变差了？换回原来的配料

然后继续试下一个改动。再试再比。50 轮下来，你的菜谱可能 9.5 次都是成功的。

**autoresearch 对你的 Skills 做的正是这件事：**

- "菜谱" = 你的 Skill 提示词
- "做菜" = 运行这个 Skill
- "品尝" = 给输出打分
- 你只需要提供"打分标准"

---

## 关键：一张 Yes/No 检查清单

整个过程中，你唯一要做的事是：定义"好"的标准。

用一个简单的 Yes/No 检查清单：

每个问题只检查输出的一个具体方面。通过或失败。就这么简单。

**为什么要用 Yes/No，而不是 1-10 分？**

因为"评分 1-10"太模糊，每次结果都不一样。而 Yes/No 是客观的、可重复的。

**举例：一个 Landing Page 文案的检查清单**

| 检查项 | 说明 |
|--------|------|
| 标题是否包含具体数字或结果？ | 捕捉模糊标题如"助力业务增长" |
| 文案是否没有使用"革命性""协同效应"等流行词？ | 避免 AI 腔 |
| CTA 是否使用了具体的动词短语？ | 捕捉弱 CTA 如"了解更多" |
| 第一行是否点出了具体的痛点？ | 捕捉通用开头如"在当今快节奏的世界..." |
| 总字数是否在 150 字以内？ | 避免冗长啰嗦 |

**3-6 个问题最合适。** 超过这个数量，Skill 会开始"应试"——就像学生背诵答案却不理解内容。

---

## 实际跑起来是什么样的？

作者用这个方法优化自己的 Landing Page 文案 Skill，结果：

**56% → 92%，4 轮改动，3 个保留，1 个撤销。**

以下是 Agent 实际对提示词做的改动：

**改动 1：** 添加了一条具体规则——"标题必须包含具体数字或结果，绝不使用模糊承诺如'改变你的业务'。"

**改动 2：** 添加了一个禁用词列表——"绝不使用：革命性、前沿、协同效应、下一层级、颠覆性、解锁、改变。"

**改动 3：** 添加了一个优质 Landing Page 示例，把痛点开场和 CTA 高亮出来，让 Skill 看到"好"长什么样。

**改动 4（被撤销）：** 尝试缩减字数限制，但因为文案变得太薄、CTA 效果变差，被系统自动撤销了。

**最后你得到：**

- 优化后的 Skill（单独保存，原始版本不受影响）
- 每轮的评分日志
- 一份改动记录，解释每个改动为什么尝试、是否有效
- 原始 Skill 的备份

---

## 这不只是用来优化 Skills

任何可以被量化评分的东西，都可以用 autoresearch：

| 应用场景 | 检查清单示例 |
|----------|-------------|
| **网页速度优化** | 页面加载时间是否 < 100ms？改动一个参数，测速，保留或撤销。有人跑了 67 轮，从 1100ms 降到 67ms。 |
| **冷邮件** | 是否提到对方公司？是否在 75 字以内？是否以具体问题结尾？让 Agent 跑 50 个变体。 |
| **Newsletter 开头** | 开头是否包含个人细节？是否没有陈词滥调？让 Agent 自动优化你的写作风格。 |

**核心原则：能评分，就能 autoresearch。**

---

## 如何开始？

**Step 1：** 下载 autoresearch Skill，放到你的 Claude Code 或 Cowork 的 skills 文件夹里。

**Step 2：** 选择一个你想优化的 Skill。说："用 autoresearch 优化我的 [Skill 名称] Skill。"——选那个让你最头疼的，那个一半时间输出很好、一半时间输出垃圾的。

**Step 3：** Agent 会问你三件事：优化哪个 Skill？用什么测试输入？检查清单问题是什么？

**Step 4：** 它运行你的 Skill，显示你的初始分数。这是基线。

**Step 5：** 它在浏览器打开一个实时仪表盘——分数曲线随时间上升，每个检查项的通过/失败情况，每次尝试的改动日志。每 10 秒自动刷新。

**Step 6：** 走开。Agent 进入循环：分析失败项 → 对提示词做一个小改动 → 再测试 → 分数上升就保留，下降就撤销。

它会持续自主运行，直到你停止它，或者连续三次达到 95%+。

你可以看仪表盘，也可以完全走开。它会自己跑。

---

## 为什么这很重要？

今天最聪明的模型，可能半年后就过时了。

但那份改动记录（changelog）——完整记录了什么有效、什么无效——是有长期价值的资产。

当更强大的模型出来时，你把这份 changelog 交给它，它能直接接上上次的进度继续优化。

**不是每次从零开始，而是持续积累"什么有效"的知识。**

---

## 立刻去试

选一个你表现最差的 Skill。启动 autoresearch。回来的时候，发现它真的变好了。

这可能是目前最接近"AI 自我进化"的实用方法。
