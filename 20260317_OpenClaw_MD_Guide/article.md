# 你的 AI 为什么总记不住你？OpenClaw 给了个方案

> 本文解读内容来源
> - 原始来源：[OpenClaw 核心 MD 文件完全指南](https://openclaw.ai)
> - 来源类型：技术博客
> - 作者：0xMomo
> - 发布时间：2026 年 3 月

---

我有个朋友，用了大半年的 Claude Code，助手已经学会了他的代码风格、知道他喜欢什么命名规范、甚至记得他项目的目录结构。

然后他换了台电脑。

一切归零。

这不是个例。ChatGPT、Cursor、Copilot，几乎所有主流 AI 助手都有这个问题——换台设备、清个缓存、或者单纯过了太久，AI 就"不认识"你了。

最近看到一个开源项目 OpenClaw，315K 的 GitHub 星，思路挺有意思：把 AI 的"脑子"变成一堆 Markdown 文件，存你本地。

---

为什么 AI 总是"健忘"，本质上是个成本和隐私问题。

商业 AI 公司不可能给你永久保留对话历史——存储成本先不说，你的聊天记录里有多少敏感信息？账号、密码、公司内部代码、个人隐私……存云端就是颗定时炸弹。

所以主流做法是：上下文窗口关了就清零。安全，但"无情"。

OpenClaw 的思路是：既然云端存不了，那就存本地。而且不是用什么数据库，就是 Markdown 文件。

这有什么好处？

你打开 VS Code 就能看。改错了 Git 回退。想试试不同的人格设定，开个分支。甚至可以把自己的"AI 助手配置"分享给团队，或者从别人那里 clone 一份已经调教好的。

AI 从一个"订阅的服务"变成了"我可以理解的文本"。

---

OpenClaw 把 AI 的认知系统拆成了 8 个 MD 文件。我看了下，设计得还挺讲究。

身份相关的有三个。SOUL.md 定义性格和工作哲学——你想让它是个严谨的技术顾问，还是个爱吐槽的创意伙伴，都在这里写。IDENTITY.md 更像是"人设卡"，名字、称呼方式、语气这类轻量信息。USER.md 则是用户画像，AI 知道在跟谁说话、什么语言、什么时区。

行为相关的有两个。AGENTS.md 算是"员工手册"，启动时做什么、什么操作需要用户授权、安全边界在哪。HEARTBEAT.md 挺有意思，定义 AI 空闲时干什么——检查邮件？扫日历？还是安静待着。

记忆相关的核心就一个：MEMORY.md。这是最关键的创新点。你的偏好、重要的上下文、AI 学到的经验，都记在这里。有意思的是，你不需要手动编辑它——日常对话里说"记住这个"，它会自动更新。

环境相关的两个。TOOLS.md 记录本机环境，浏览器路径、设备信息之类，这样分享 Skills 的时候不会泄露自己的基础设施配置。BOOTSTRAP.md 是初始化脚本，首次启动时自动创建目录结构、安装依赖。

---

这套设计有个细节我觉得挺聪明：MEMORY.md 不是每次都全量加载。

长期记忆可能会变得很大。OpenClaw 用了一个 memory_search 的机制，按需检索。这样既控制了 token 消耗，响应速度也有保障。

另一个细节是透明度。你想知道 AI 记住了你什么？打开文件看。想删掉某条记忆？直接删。没有任何黑盒。

而且因为是 Git 管理的纯文本，你可以看到 AI 的"学习轨迹"——它什么时候开始有这个偏好的？什么时候学会这个规则的？commit history 里都有。

---

如果只是偶尔用 ChatGPT 问问题，这套东西可能过于复杂。

但如果你想让 AI 真正"认识"你，或者你是开发者想定制自己的 AI 工作流，这个"文件即记忆"的设计值得看看。

GitHub 搜 `openclaw/openclaw`，或者国内访问 `open-claw.org.cn`。

安装完后，`~/.openclaw/workspace` 目录下就是你的 AI 助手的"大脑"。打开看看，可能比你想象的更简单。
