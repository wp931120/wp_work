# 本地知识库搜索新方案：QMD 让 AI 真正读懂你的笔记

> 完全离线、混合搜索、AI 重排序，这个开源项目让个人知识库检索进入新阶段

---

## 一、为什么你需要本地搜索？

想象一下这个场景：

你半年前写了一篇关于"React 性能优化"的笔记，里面提到了几个关键技巧。今天开会时老板突然问起，你打开笔记软件，却发现自己根本记不起当时把内容存在了哪个文件夹。

**更糟糕的是**：你用了 Notion、Obsidian、印象笔记……工具换了一个又一个，但搜索功能始终差强人意。要么只能匹配关键词，要么需要把数据上传到云端。

**隐私和效率，难道真的无法兼得吗？**

今天介绍的这个开源项目，或许能给你一个满意的答案。

## 二、QMD 是什么？

**QMD**（Query Markup Documents）是一个**完全本地的命令行搜索工具**，专为 Markdown 文件设计。它的核心能力可以用三个词概括：

- **混合搜索**：同时支持关键词搜索和语义搜索
- **AI 增强**：用本地 LLM 做查询扩展和结果重排序
- **零隐私泄露**：所有数据、所有模型都跑在你的机器上

这个项目由 Tobi Lutke（Shopify 创始人）开发，目前已经在 GitHub 上获得了 **15000+ stars**，并且保持活跃更新。

---

## 三、核心技术：三层搜索架构

QMD 的搜索流程分为三个阶段，每一层都在提升搜索结果的质量。

### 第一层：BM25 关键词搜索

这是最快的一层，使用传统的全文检索技术。当你搜索"React 性能"时，它会找出所有包含这两个词的文档。

**优点**：速度极快，毫秒级响应
**局限**：无法理解语义，"性能优化"搜不到"提速技巧"

### 第二层：向量语义搜索

这一层使用嵌入模型（Embedding）将文本转换为向量，然后计算语义相似度。

**关键突破**：即使你的查询词和文档内容不完全匹配，只要语义相近就能被找到。比如搜索"怎么让页面加载更快"，可以找到标题为"React 渲染优化指南"的文档。

### 第三层：LLM 重排序

这是 QMD 的"杀手锏"。前两层会召回约 30 篇候选文档，然后用一个本地部署的 Reranker 模型（Qwen3-Reranker-0.6B）对它们进行精细排序。

**效果**：最终返回的结果质量大幅提升，真正做到了"懂你所问"。

## 四、实际使用体验

### 安装与配置

```bash
# 全局安装
npm install -g @tobilu/qmd

# 添加你的笔记目录为"集合"
qmd collection add ~/notes --name notes

# 生成向量索引（首次需要几分钟）
qmd embed
```

### 三种搜索模式

```bash
# 1. 快速关键词搜索
qmd search "React 性能优化"

# 2. 语义搜索（推荐）
qmd vsearch "怎么提升页面渲染速度"

# 3. 混合查询（质量最高）
qmd query "React 应用有哪些性能优化手段"
```

**第三种模式**会自动进行查询扩展，生成多个变体问题，然后综合所有结果，用 AI 重排序后返回最佳答案。

## 五、为什么选择本地方案？

### 隐私安全

你的会议记录、个人笔记、工作文档……这些敏感数据不需要上传到任何云端服务器。所有索引、搜索、AI 推理都在本地完成。

### 离线可用

没有网络？没关系。QMD 依赖的模型都会下载到本地缓存，断网环境下依然可以正常工作。

### 可定制化

你可以选择自己喜欢的嵌入模型和重排序模型，调整索引策略，甚至对模型进行微调以适应特定领域的术语。

## 六、进阶玩法：AI 智能体集成

QMD 支持 **MCP（Model Context Protocol）**，可以直接与 Claude Desktop 或 Claude Code 集成。

**典型场景**：

1. 你在对话中问 Claude："帮我总结一下之前关于 React 性能优化的笔记"
2. Claude 通过 MCP 调用 QMD，检索你的本地知识库
3. 获取相关文档后，生成精准的总结回答

这相当于给你的 AI 助手装上了"长期记忆"，让它真正了解你的工作上下文。

## 七、技术架构解析

```
用户查询 → 查询扩展（LLM）→ [原始问题 + 多个变体]
                    ↓
         ┌──────────┼──────────┐
         ↓          ↓          ↓
    BM25 搜索   向量搜索    （对每个变体执行）
         ↓          ↓
         └────┬─────┘
              ↓
    RRF 融合（原始结果×2 权重）
              ↓
    Top 30 候选 → LLM 重排序
              ↓
    位置感知融合 → 最终结果
```

**核心依赖**：
- `better-sqlite3`：本地数据库存储
- `node-llama-cpp`：本地 LLM 推理
- `sqlite-vec`：向量相似度搜索
- `@modelcontextprotocol/sdk`：AI 智能体集成

**默认模型**（自动下载）：
- 嵌入模型：embeddinggemma-300M（约 300MB）
- 重排序模型：qwen3-reranker-0.6b（约 640MB）
- 查询扩展：qmd-query-expansion-1.7B（约 1.1GB）

## 九、写在最后

QMD 代表了一个趋势：**AI 能力正在从云端下沉到本地**。

过去，高质量的语义搜索需要调用昂贵的云端 API；现在，一台普通笔记本电脑就能运行完整的混合搜索 pipeline。

**更重要的是**，这个项目让我们看到了"隐私优先"的 AI 应用应该如何设计——不牺牲用户体验，但把数据控制权交还给用户。

如果你也在寻找一个靠谱的个人知识库搜索方案，QMD 值得尝试。
