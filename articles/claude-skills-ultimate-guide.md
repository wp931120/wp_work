## 01 Claude Skills 到底是什么？

**一句话：Claude Skills 是保存为 Markdown 文件的预加载指令集。**

在任何 Claude 聊天或项目中，你可以调用一个 Skill，Claude 会立即执行你创建的指令。**不用反复解释，不用重新提示，不用每次新对话都复制粘贴上下文。**

这样理解：

现在，每次你打开新的 Claude 聊天，都是从零开始。Claude 对你、你的声音、你的标准、你喜欢的事情完成方式一无所知。

**Skill 改变了这一点。** 它把 Claude 需要知道的所有内容打包成一个可复用的文件，随时调用。

**一个实际例子：**

想象一个"品牌声音"Skill，包含你公司的一切（语气、风格、受众、关键信息、优秀文案示例）。

不用每次都解释所有这些，你只需要说：

> "Hey Claude，用我的品牌声音 Skill 写一条关于 [X] 的 LinkedIn 帖子。"

**可能性真的无限。** 写作 Skill、调研 Skill、语法检查器、外联模板 Skill……任何重复性的、上下文和指令很重要的任务，都是 Skill 的候选。

## 02 如何构建和使用 Skills

构建一个 Skill 只需不到 30 分钟，而且**只需做一次**，就能为你的生产力创造一个永久性资产。

### Step 1：启用 Skill-Creator

进入 **Customize → Skills**，确保启用了 "Skill-Creator"。

这是一个用来构建 Skills 的 Skill（来自 Anthropic 官方）。

### Step 2：提示 Claude

启用后，告诉 Claude：

> "我想为 [工作流] 构建一个 Skill，帮我建。"

**这是从零构建第一个 Skill 的模板提示词：**

```
你正在构建一个 Claude Skill——一个 Markdown 格式的可复用指令集。

我的 Skill 用于：[描述任务——例如写 X 类文章、检查语法、研究主题]

这是 Claude 需要知道的上下文：
- 我的名字/品牌：[name]
- 我的受众：[你为谁写作]
- 我的语气和声音：[你希望 Claude 听起来如何]
- 我的标准：[好的输出是什么样的]
- 要避免什么：[常见错误、永远不用的词、格式]

使用这些上下文，写一个完整的 SKILL.md 文件：
1. 用一句话描述这个 Skill 做什么
2. 定义 Claude 在这个 Skill 激活时扮演的角色
3. 列出 Claude 必须遵守的确切规则
4. 包含至少一个优秀输出的示例
5. 最后加一个质量检查清单，Claude 在响应前自检

格式化为正确的 Markdown 文件，我可以直接保存并上传到 Claude Project。
```

### Step 3：保存 Skills

Claude 构建完成后，会输出一个 zip 和/或 Markdown 文件。点击 **"Copy to Skills"** 保存到你的 Anthropic 账户。

我也喜欢把 Skills 下载到笔记本电脑（专用文件夹），但这是可选的。

回到 **Customize → Skills → My Skills**，确保你的新 Skill 存在且已启用。

### Step 4：使用你的 Skill

现在，只需提示 Claude 使用你的 Skill：

> "用我的 [x] Skill 来做 [x]。"

就这么简单！

## 03 如何构建**优秀**的 Skills

任何人都能构建 Claude Skills，但很少有人能真正构建出**有用**的优秀 Skills。

以下是我的建议（基于几个月经验）：

### 🔄 反向提示

告诉 Claude：

> "我想构建 [x] Skill，问我 10-50 个问题来帮你构建它。"

### 🔁 反向构建

告诉 Claude：

> "基于你对我的一切了解，什么 Skills 对我有帮助？"

### 📚 上下文越多越好

提供尽可能多的上下文。扔 PDF、附加文档——任何你有的东西。

### 💬 用聊天记录当上下文

使用你现有的聊天记录作为上下文。（例如：把这个聊天里的一切都变成 Skill）

我就是这么做的"语法检查器"Skill，它立刻就知道了我想要的一切，因为我们已经聊了几个月。

### 🔄 迭代

把 Claude 的第一个 Skill 输出当作**初稿**。读完整文件，写下你想改的地方，提示 Claude 修改。

我经常创建 3 个 Skills 才能得到 100% 满意的结果。

### ✨ 包含真实示例

**提升 Skill 输出质量的最快方法：粘贴一个优秀示例。**

Skill 文件里的一个强示例，抵得上十条指令。

### ✅ 质量检查清单

每个 Skill 结尾加一个简短清单。3-5 个问题，Claude 在输出前自检：

- 这符合语气吗？
- 我避免禁用词了吗？
- 这是正确的格式吗？

### 👀 手动审查

虽然痛苦，但**一定要手动审查 Skills**。是的，需要几分钟手工劳动，但结果是一个从此为你节省时间的 Skill。

## 04 Claude Skills 2.0：三月重大更新

Anthropic 在三月发布了 Skills 框架的重大更新。

三个新功能值得了解：

### 1️⃣ 内置评估和测试

**更新前：** 没有可靠方法知道你的 Skill 是否真的有效。你建一个，试几次，觉得还行，就继续了。如果后来出了问题（比如模型更新后），你往往要等到交付物出错才发现。

**Skills 2.0 修复了这个问题。**

保存 Skill 到账户之前，你就已经知道输出实际会是什么样子。

你写几个真实的提示（你真正会问 Claude 的那种），Claude 跑两次：一次加载你的 Skill，一次不加载。两个输出都根据你定义的断言评分，然后在查看器中显示，你审查实际输出并留下反馈。

### 2️⃣ A/B 测试

A/B 测试允许你用相同的提示比较同一 Skill 的两个版本。

改了你的"品牌声音"Skill，想知道是否真的提升了输出？

现在你可以跑 A/B 测试。

### 3️⃣ 触发器优化

**一个 Skill 无论内部指令多好，如果触发时机不对或根本不触发，就是坏的 Skill。**

Skills 2.0 包含一个自动化流程，重写并测试 Skill 描述的不同版本，直到找到可靠触发的版本。

这消除了"你说'用 [x] Skill'但 Skill 不触发"的问题。

**使用这些新功能：**

打开加载了 Skill 的 Claude 聊天，告诉 Claude 你想做什么。

示例：
- "对这个 Skill 跑评估。"
- "A/B 测试这两个版本。"
- "优化我的触发器描述。"

## 05 真实工作流和超级提示

现在你知道如何构建和优化 Skill，这里有一些真实工作流展示真正的可能性。

### 🎯 1. 品牌声音 Skill

用于创建声音/语气/写作风格。

包含你的品牌调性、目标受众、常用词汇、禁用词、优秀示例。

### 📄 2. PDF 生成器 Skill

把任何文本转成格式良好的 PDF。

### 📝 3. 文档摘要器 Skill

几秒钟内总结任何文本。

### 其他值得构建的工作流：

| Skill 类型 | 用途 |
|-----------|------|
| **ELI5** | 用简单术语解释复杂主题（粘贴主题/文章/文本，触发 Skill） |
